# 편의점 최적입지 스코어링 및 검증 (서울시 중랑구 기준)

서울 전체 상권·유동인구 공공데이터로 편의점 매출 결정 요인을 학습하고, 중랑구를 100m 격자 단위로 스코어링해 출점 후보지를 도출·검증한 개인 포트폴리오 프로젝트입니다.

## 배경

졸업작품에서는 입지 선정 가중치를 선행연구 값으로 그대로 가져다 썼습니다. 이 프로젝트는 그 한계를 "가중치를 데이터로 직접 학습"하는 방식으로 발전시킨 것입니다. 서울 전체 편의점 매출 데이터로 LightGBM 모델을 학습하고, 그 모델을 중랑구 100m 격자에 적용해 점수화한 뒤, 점수 × 기존 점포 유무 4분면 분석과 면목2동 동부시장 인근 현장 임장으로 결과를 검증합니다.

## 핵심 결과 요약

- 서울시 편의점 9,407개(정제 후) 기준, 상권 712개에 대해 LightGBM 매출 예측 모델을 학습했습니다.
- 절대 매출액 예측력(R²)은 높지 않지만(테스트 R² 0.02~0.11), 이 프로젝트의 실제 목적인 **순위 기반 스코어링**에서는 스피어만 순위상관 0.31, 실제 매출 상위 20% 상권 중 46.4%를 모델도 상위 20%로 짚어내는 적중률을 확인했습니다.
- 중랑구 100m 격자 1,858개에 모델을 적용해 "고득점·점포없음(신규 후보지)" 806개를 포함한 4분면 분류를 완료했습니다.
- 개발 과정에서 발견한 데이터 이슈(생활인구-집계구 코드 기준연도 불일치, CSV 컬럼 밀림 버그, 면적배분 공식 버그 등)를 모두 근본 원인까지 추적해 수정했습니다 — 아래 "방법론 하이라이트" 참고.

## 파이프라인

```
Phase 1  데이터 수집         data/raw에 원본 데이터 확보, 데이터 명세서 작성
Phase 2  정제·통합           좌표계 통일(EPSG:5179), 편의점 필터링, 생활인구 집계
Phase 3  격자·피처 계산      서울 100m 격자 생성, 경쟁밀도·접근성·생활인구·집객시설 피처
Phase 4  모델 학습(서울 전체) 타깃 설계, LightGBM 학습, v1→v3a 반복 개선
Phase 5  중랑구 스코어링      100m 격자 점수화, 4분면 검증, 인터랙티브 웹지도
Phase 6  README·패키징       (현재 문서)
```

각 단계는 `src/00_*.py` ~ `src/22_*.py` 번호 순서로 실행되며, 스크립트별 상세 로그는 `outputs/`에 저장됩니다.

## 방법론 하이라이트

**좌표계 통일**: 분석용 EPSG:5179, 웹지도 출력용 EPSG:4326으로 통일 (`src/utils_geo.py`).

**생활인구-집계구 코드 문제 해결**: 생활인구 집계구코드와 최신 집계구 경계(`BND_TOTAL_OA_PG`, 2024년 개편본)의 매칭률이 0%였습니다. 처음엔 "행정안전부 코드 vs 서울시 코드" 체계 차이로 추정했지만, 재조사 결과 진짜 원인은 **기준 연도 불일치**였습니다 — 서울시 생활인구는 2016년 4분기 기준 집계구 코드를 그대로 씁니다. SGIS 승인자료(`bnd_oa_11_2016_4Q`)와 대조해 **100% 매칭**(`pandas.merge(validate='one_to_one')`로 엄격 검증)을 확인하고, 집계구 해상도를 그대로 유지할 수 있었습니다. (자세한 조사 경위: [데이터명세서.md](docs/데이터명세서.md) 6·11·12번 항목)

**면적 비례 배분**: 집계구 인구를 100m 격자에 배분할 때, "겹친 면적으로 가중평균"을 냈다가 격자가 집계구 일부만 덮어도 인구 전체를 배정하는 버그를 발견해 "겹친 면적 비율만큼 비례 배분·합산"으로 수정했습니다. 격자 합산 인구가 원본 집계구 총합과 0.00% 오차로 일치함을 검증했습니다.

**모델 반복 개선(v1 → v3a)**: 상권 총매출을 그대로 타깃으로 쓴 v1에서, 상권 규모 효과를 지우려고 "점포당 평균매출"로 바꾼 v2를 시도했으나 오히려 성능이 떨어졌습니다. 원인을 "타깃 재정의 효과"와 "피처 추가 효과"로 분리한 v3a/v3b 진단을 통해, 저조했던 원인이 점포 수 적은 상권의 타깃 노이즈였음을 밝히고, 최종적으로 **v3a(타깃=총매출 유지, 피처=집객시설 포함 12종)**를 채택했습니다.

## 모델 및 피처

**최종 모델(v3a)**: LightGBM, 상권 708개 × 피처 12종(경쟁밀도 3, 접근성 3, 생활인구 4, 집객시설 2)

| 지표 | 값 |
|---|---|
| 5-Fold CV R² | 0.203 (±0.093) |
| 테스트셋 R² | 0.018 |
| 테스트셋 스피어만 순위상관 | 0.312 |
| 테스트셋 Top20% 적중률 | 46.4% |

절대 매출액을 정확히 맞추는 모델은 아닙니다 (위치 피처만으로는 매출 분산의 일부만 설명되며, 나머지는 매장 운영·브랜드 등 저희가 갖지 않은 요인에서 옵니다). 다만 이 프로젝트의 목적인 "어디가 상대적으로 더 좋은 입지인가" 줄 세우기에는 유의미한 신호를 보입니다.

**Feature Importance 상위**: 주간_평일_평균인구 > 지하철 월간 승하차량 > 반경 100m 경쟁점 수 > 반경 300m 경쟁점 수. 흥미롭게도 경쟁점 수는 매출과 **양의 상관관계**(밀집 지역이 오히려 우수 상권)를 보였습니다.

v1부터 v3b까지 전 버전의 비교는 `outputs/20_v1_v2_v3a_v3b_comparison.csv`에 있습니다.

## 중랑구 스코어링 결과

v3a 모델을 중랑구 100m 격자 1,858개에 적용하고, 격자 내 기존 편의점 유무와 교차한 4분면 분류 결과:

| 분면 | 격자 수 |
|---|---|
| 고득점 · 점포없음 (신규 후보지) | 806 |
| 저득점 · 점포없음 | 783 |
| 저득점 · 점포있음 | 143 |
| 고득점 · 점포있음 (검증된 자리) | 126 |

CLAUDE.md에 명시된 현장 검증 지점(면목2동 중랑동부시장) 반경 300m에서는, 기존 점포가 있는 격자들이 실제로 고득점으로 나와 모델이 이미 성공한 상권을 알아본다는 것을 확인했습니다.

**인터랙티브 웹지도**: [outputs/22_jungnang_map.html](outputs/22_jungnang_map.html) — 브라우저로 열면 4분면별 색상 격자, 기존 편의점 위치, 동부시장 마커를 레이어별로 확인할 수 있습니다.

## 한계

- **집계구 내 균등분포 가정**: 생활인구를 격자로 배분할 때 집계구 안에서는 인구가 고르게 분포한다고 가정했습니다. 실제로는 건물·도로 배치에 따라 다를 수 있습니다.
- **낮은 절대 예측력**: 위치 피처 12종만으로는 매출 분산의 대부분을 설명하지 못합니다. 매장 크기, 브랜드, 운영 방식 등 미보유 요인의 영향이 클 것으로 추정됩니다.
- **학습-테스트 성능 격차**: v3a는 학습셋 R²(0.54)과 테스트셋 R²(0.02) 격차가 커서, 절대 금액 예측보다 순위 비교 용도로만 신뢰하는 것이 안전합니다.
- **표본 크기**: 상권 단위 학습 표본이 700여 개로 크지 않아, 성능 지표 자체에도 표본오차가 있습니다.
- **2016년 기준 집계구 경계**: 생활인구 조인에 쓴 집계구 경계가 2016년 시점이라, 이후 신축·재개발된 지역의 실제 물리적 경계와는 차이가 있을 수 있습니다.
- **행정동/집계구 경계 기준일자 불일치**: `BND_ADM_DONG_PG`(2025-06)와 `BND_TOTAL_OA_PG`(2024-06) 사이 1년 차이로 인한 잠재적 코드 불일치 가능성.

## 실행 방법

```bash

source venv/Scripts/activate   # Windows Git Bash 기준

# data/raw에 원본 데이터를 준비한 뒤, 순서대로 실행
python src/00_inspect_raw_data.py
python src/01_reproject_to_5179.py
# ... 02번부터 22번까지 번호 순서대로 ...
python src/22_map_jungnang.py
```

필요 라이브러리는 `requirements.txt` 참고 (`pip install -r requirements.txt`).

## 라이선스 및 출처 고지

이 프로젝트에서 사용한 `bnd_oa_11_2016_4Q`, `bnd_oa_11_2015_4Q` 집계구 경계 데이터는 **SGIS(통계지리정보서비스)**에서 자료 신청·승인을 받아 취득했습니다. 재사용 시 SGIS 출처 표기가 필요합니다. 그 외 데이터는 서울열린데이터광장, 국가데이터처, 소상공인시장진흥공단이 공개한 공공데이터를 사용했습니다.
